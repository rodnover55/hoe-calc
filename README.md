# Калькулятор урона — Heroes of Might and Magic: Olden Era

Веб-приложение для расчёта урона в бою: слева характеристики атакующего,
в центре параметры самой атаки (удача, дистанция, модификаторы), справа —
защищающегося.

Характеристики можно заполнить вручную или выбрать юнита из пресетов:
фракция (Храм, Некрополис, Сильван, Подземелье, Улей, Раскол, Нейтралы) →
юнит → грейд (база или одно из двух альтернативных улучшений). При выборе
подставляются статы юнита и показывается его портрет.

Героя каждой стороны тоже можно выбрать из базы (фракция → герой или
поиск по имени): подставляются его стартовые атака, защита, сила магии
и знание (дальше они правятся вручную), показываются портрет, класс и
специализация. Стартовые навыки героя попадают в редактируемый список:
навык можно добавить из каталога, поменять уровень мастерства, выбрать
поднавыки или удалить. Уровень героя задаётся полем и масштабирует
специализацию и урон удара героя.

## Формула

```
урон = кол-во × урон существа × (20 + ATK) / (20 + DEF)
       × общие модификаторы × типовые модификаторы (мин. 10%) × удача
       × дальность × режим
```

- Удачный удар ×1.5, неудачный ×0.5.
- Итог всегда наносит минимум 1 урона.

Режим атаки выбирается кнопками и строится по способностям выбранного
юнита (карта слагов — в `src/abilityEffects.ts`):

- Стрелок (`ranged_attack`): дальняя атака теряет 10% за каждый гекс сверх
  трёх (максимум −50%); вплотную бьёт с половинным уроном. «Снайпер»
  (`sharpshooter`) стреляет без штрафа дальности, «Дуэлянт» (`duelist`)
  бьёт вплотную без штрафа.
- Боевые стойки (`fighting_style_*`) и «Пронзающий выстрел»
  (`piercing_shot`) — альтернативные атаки с множителем ×0.5.
- Активные способности с собственным уроном («Взор Бездны», «Ледяное
  дыхание», «Звездопад», «Мгновенная кара» и другие) — отдельные режимы:
  урон считается по формуле из описания способности (доля урона обычной
  атаки или `база + множитель × численность`), без АТК/ЗЩТ,
  модификаторов, дальности и удачи. Магический урон снижает «Защита от
  магии» цели, чистый не снижается ничем.
- Постоянные защитные способности цели — «Защита от выстрелов»
  (`ranged_defence_*`), «Защита в ближнем бою» (`melee_defence_*`),
  «Презрение» (`disdain`) — автоматически уменьшают урон
  соответствующего типа атаки (учитываются как типовые модификаторы).
- «Драконий клинок» (`draconic_blade`) удваивает урон ближней атаки.
- Ответный удар проставляется автоматически: дальняя атака, атака через
  гекс (`long_reach`, стойка-плевок `fighting_style_viscous_spit`),
  активные способности и «Стремительный удар» (`swift_strike`) ответа не
  провоцируют; галочку можно переключить вручную.
- «Двойной удар» (`double_strike`) и «Двойной выстрел» (`double_shot`,
  только при стрельбе) добавляют второй удар: он наносится после ответа
  выжившими атакующими по остатку отряда защитника. Удача в калькуляторе
  действует на оба удара одинаково — в игре она выпадает на каждый удар
  отдельно. На активные способности двойной удар не распространяется.

Прочие эффекты способностей (дебаффы и DoT от атак, площадь стоек и
мультицельных атак, ауры, временные баффы, условные бонусы вроде «+50%
урона целям выше рангом») в расчёт не входят — способности юнита
показываются списком в его карточке.

Данные — официальная вики игры; игра в раннем доступе, цифры могут меняться.

## Герои

Карта эффектов героев — в `src/heroEffects.ts`. Бонусы складываются с
полями формы в момент расчёта и не перезаписывают введённые вручную
значения; что учтено, показывается заметками под карточкой героя.

- Специализации по существам («Соль земли» у мечников и т.п.): +N к
  атаке и защите своего отряда этого существа за каждые K уровней героя;
  такие же существа противника теряют столько же. Бонусы скорости,
  инициативы и здоровья из этих специализаций в расчёт не входят.
- «Стрелок»/«Боец»: +% урона обычными атаками (растёт с уровнем, с
  добавкой по типу удара); «Защита»: −% входящего урона. Учитываются как
  типовые модификаторы и не действуют на способности с собственным
  уроном; проценты защитника не усиливают его ответный удар — формула
  не учитывает модификаторы в ответе.
- «Истинный лорд»: +% от атаки и защиты героя к атаке и защите существ.

### Навыки

Навыки героя считаются по редактируемому списку в его карточке: при
выборе героя туда сеются его стартовые навыки (уровень мастерства — из
слага, `basic_`/`advanced_`/`expert_`), дальше список правится вручную —
«Добавить навык» (из выпадающего списка или поиском по названию, в том
числе по имени уровня — «нападение» находит «Атаку»), выбор уровня,
чекбоксы поднавыков (доступны с продвинутого уровня — по три на
продвинутом и экспертном), удаление. Список хранится в share-ссылке;
старые ссылки без навыков при открытии получают стартовые навыки героя,
поэтому считаются как раньше.

Влияют на расчёт (числа парсятся из русских описаний каталога):

- Уровни навыков: «Нападение» +10/15/20% урона обычными атаками,
  «Оборона» −10/15/20% входящего, «Сопротивление» −15/25/35%
  магического урона способностей, «Ратное дело» +10/15/20 к урону удара
  героя, «Боевая магия» — атака/защита существ +15/20/25% от силы магии
  и знаний героя (округление вниз).
- Поднавыки: «Стрельба» +15% урона выстрелами и на расстоянии,
  «Укрытие» −20% от них же, «Мастерство ближнего/дальнего боя» ±10% по
  типу удара, «Теневые клинки» +1 урона своим, «Щиты и панцири» −1
  урона существам противника, «Непоколебимость»/«Неостановимая
  сила»/«Час волка» — процентные штрафы атаке и защите существ
  противника (от статов формы, округление вниз), «Договор с
  волшебниками» −10% магического урона, «Хроническая слабость» −1 к
  здоровью существ противника, «Фехтование на мечах» и «Бесконечная
  харизма» — плоские атака и защита, «Авторитет боевого мага» и
  «Авторитет мага-защитника» — проценты от силы магии и знаний,
  «Элитные стражи» (существа 1–3-го рангов) и «Бродячая армия»
  (нейтралы) — проценты от атаки и защиты героя, «Усиленные тренировки»
  +2 к здоровью своих, «Практичные заклинания» +10 к удару героя.
- «Бонус удваивается, если герой владеет «X»» учитывается, когда навык
  X есть в списке той же стороны.

Процентные бонусы урона действуют только на обычные атаки и не
усиливают ответный удар (формула не учитывает модификаторы в ответе);
плоские прибавки и штрафы статам действуют через характеристики и
потому влияют и на ответ. Прочие навыки и поднавыки (магия школ,
логистика, лидерство, удача, эффекты от боевого духа и соседних отрядов
и т.д.) на расчёт не влияют; заметки под карточкой героя перечисляют
только применённые эффекты.

### Удар героя

Когда у атакующего выбран герой, в списке режимов атаки появляется
«Удар героя» с уроном в подписи. Формула из игровых файлов (v0.80.31):

```
урон = 30 × (1 + 0.2 × (уровень − 1)) + бонус специализации + бонус навыков
```

Удар героя — чистый урон: атака героя, защита цели и её защитные
способности не участвуют, ответного удара нет. Специализации на удар
героя (Стингер, Толкет, Старый лорд Мандалл, Курсон) учитываются по
числам из описания; из навыков учитываются «Ратное дело» (+10/15/20) и
поднавык «Практичные заклинания» (+10), артефакты — нет.

## База юнитов

Каждый юнит — YAML-файл в `src/data/units/<фракция>/<slug>.yaml`:

```yaml
id: warden_gnat
name: "Страж-мошка"
nameEn: "Warden Gnat"
faction: hive
tier: 1
grade: 1            # 0 — база, 1–2 — альтернативные улучшения
upgradeOf: gnat     # только у улучшений
image: units/hive/warden_gnat.webp
stats:
  health: 9
  attack: 3
  defense: 6
  damageMin: 2
  damageMax: 4
  initiative: 5
  speed: 5
attackType: long_reach   # melee | long_reach | ranged
flying: false
abilities:               # все способности юнита с olden-era.com
  - id: long_reach       # слаг — имя файла иконки на сайте
    name: "Атака на расстоянии"
    nameEn: "Long Reach"
    description: "Атакует отряд противника через один гекс. Не провоцирует контратаку и не занимает гекс рядом с целью."
    descriptionEn: "Has an attack that targets the hex directly behind an adjacent hex. It doesn’t provoke counterattacks and doesn’t move the attacker next to their target."
growth: 6
cost: 24
source: https://www.olden-era.com/ru/units/warden_gnat
```

Портреты лежат в `public/units/<фракция>/`. Файлы загружаются в приложение
через `import.meta.glob` в `src/units.ts`.

Данные и картинки собраны с [olden-era.com](https://www.olden-era.com)
(фанатская база по игре). Обновить базу целиком:

```bash
npm run update-units
```

Иконки в шапках колонок удачи лежат в `public/stats/` — это `/img/stats/luck.webp`
(клевер) и `/img/stats/attack.webp` (мечи) оттуда же; отдельной иконки неудачи в
игре нет, поэтому колонка неудачи показывает тот же клевер, обесцвеченный через CSS.

## База героев

Каждый герой — YAML-файл в `src/data/heroes/<фракция>/<slug>.yaml`:

```yaml
id: niev
name: "Нив"
nameEn: "Niev"
faction: hive
class: "Enforcer"        # сайт не переводит классы
classEn: "Enforcer"
image: heroes/hive/niev.webp
stats:                   # стартовые характеристики 1-го уровня
  attack: 4
  defense: 0
  spellPower: 2
  knowledge: 1
specialization:
  id: shooter            # слаг английского названия
  name: "Стрелок"
  nameEn: "Shooter"
  description: "…"       # полный текст с числами — источник данных
  descriptionEn: "…"     # для карты эффектов в src/heroEffects.ts
skills:                  # стартовые навыки со страниц /skills/<slug>
  - id: basic_offence
    name: "Основы нападения"
    nameEn: "Basic Offense"
    description: "…"
    descriptionEn: "…"
source: https://www.olden-era.com/ru/heroes/niev
```

Портреты лежат в `public/heroes/<фракция>/`, загрузка — через
`import.meta.glob` в `src/heroes.ts`. Обновить базу героев:

```bash
npm run update-heroes
```

## База навыков

Каждый навык — YAML-файл в `src/data/skills/<slug>.yaml` (слаг без
префикса уровня мастерства):

```yaml
id: offence
name: "Атака"            # базовое имя из JSON-LD страницы навыка
nameEn: "Offence"
levels:                  # ровно три уровня; индекс — уровень − 1
  - name: "Основы нападения"
    nameEn: "Basic Offense"
    description: "…"     # полный текст с числами — источник данных
    descriptionEn: "…"   # для карты эффектов в src/heroEffects.ts
  - …                    # продвинутый
  - …                    # экспертный
subskills:               # только свои поднавыки; синергии чужих не пишутся
  - id: archery
    tier: 2              # минимальный уровень мастерства: 2 или 3
    name: "Стрельба"
    nameEn: "Archery"
    description: "…"
    descriptionEn: "…"
source: https://www.olden-era.com/ru/skills/basic_offence
```

Загрузка — через `import.meta.glob` в `src/skills.ts`. Обновить каталог
(страницы `/skills/{basic|advanced|expert}_<slug>` обоих языков; свои
поднавыки уровня — первые N элементов сетки, где N берётся из JSON-LD
«Sub-skills / Perks Count»):

```bash
npm run update-skills
```

У пары навыков на сайте нет русского перевода (базовое «Ратное дело»,
базовый «Шёпот стихий», «Магия дня») — в `description` тогда лежит
английский текст; парсер бонуса удара героя понимает оба варианта.

## Локализация

Кнопка в шапке переключает язык интерфейса (выбор запоминается в
localStorage; при первом заходе берётся язык браузера). Поиск юнитов
работает по названиям на всех языках сразу.

Каждый язык — отдельный YAML-файл в `src/locales/` (`ru.yaml`,
`en.yaml`), подхватывается автоматически через `import.meta.glob`:

```yaml
code: de           # код языка
label: DE          # подпись на кнопке переключателя
locale: de         # BCP-47: формат чисел и множественное число
strings:           # секции строк; ключи — как в ru.yaml
  app:
    title: "…"
  …
```

Чтобы добавить язык, скопируйте `ru.yaml`, переведите строки и поправьте
`code`/`label`/`locale`. Русская локаль — эталонная: недостающие в новом
файле ключи берутся из неё. Названия юнитов и способностей приходят из
базы юнитов (`name`/`nameEn`), а не из локали: не-русские языки
показывают английские названия.

## Разработка

```bash
npm install
npm run dev      # dev-сервер с HMR
npm run build    # проверка типов + продакшен-сборка в dist/
npm run preview  # локальный просмотр собранного приложения
```

Стек: React 19, TypeScript, Vite. Логика расчёта — в `src/formula.ts`,
карта способностей и режимов атаки — в `src/abilityEffects.ts`, карта
эффектов героев — в `src/heroEffects.ts`, каталог навыков — в
`src/skills.ts`, интерфейс — в `src/App.tsx`, локализация — в
`src/i18n.ts` и `src/locales/`.
